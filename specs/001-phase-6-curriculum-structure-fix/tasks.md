## Dependencies & Execution Order

### Phase Dependencies

- **Setup & Analysis (Phase 1)**: No dependencies - can start immediately
- **Directory Reorganization (Phase 2)**: Depends on Phase 1 completion
- **Sidebar Configuration (Phase 3)**: Depends on Phase 2 (files must be in place)
- **Curriculum Ingestion (Phase 4)**: Depends on Phase 2 (files must be in place)
- **Validation & Testing (Phase 5)**: Depends on Phase 3, Phase 4 completion
- **Polish & Documentation (Phase 6)**: Depends on all previous phases

### User Story Dependencies

- **User Story 1 (Structure)**: No dependencies - foundational reorganization
- **User Story 2 (Navigation)**: Depends on US1 - sidebar needs correct file paths
- **User Story 3 (Ingestion)**: Depends on US1 - ingestion needs correct file paths
- **User Story 4 (Validation)**: Depends on US2, US3 - can't test without sidebar + ingestion

### Within Each User Story

- Directory creation tasks (T004-T007) can run in parallel
- File move tasks (T008-T011) can run in parallel after directories created
- Frontmatter updates (T012-T015) can run in parallel
- Sidebar updates (T017-T021) should be done sequentially to avoid merge conflicts
- Ingestion tasks (T024-T027) must be sequential (config → run → verify)
- Validation tests (T028-T034) can run in parallel

### Parallel Opportunities

- **Phase 1**: All analysis tasks [P] can run in parallel
- **Phase 2**: 
  - Directory creation (T004-T007) can run in parallel
  - File moves (T008-T011) can run in parallel
  - Frontmatter updates (T012-T015) can run in parallel
- **Phase 3**: Sidebar decision (T016) is blocking, then updates can be batched
- **Phase 5**: All validation tests can run in parallel once prerequisites complete

## Implementation Strategy

### MVP First (Directory Reorganization Only)

1. Complete Phase 1: Setup & Analysis
2. Complete Phase 2: Directory Reorganization
3. **STOP and VALIDATE**: Verify files exist in correct locations
4. Commit the reorganization separately

### Incremental Delivery

1. Phase 2: Directory reorganization → Test file locations → Commit
2. Phase 3: Sidebar updates → Test navigation → Commit
3. Phase 4: Ingestion → Test Qdrant indexing → Commit
4. Phase 5: Validation → Test chatbot responses → Document results

### Parallel Team Strategy

With multiple developers:

1. Developer A: Phase 2 (Directory reorganization + frontmatter updates)
2. Developer B: Phase 3 (Sidebar configuration + build verification)
3. After both complete: Developer A → Phase 4 (Ingestion), Developer B → Phase 5 (Validation)
